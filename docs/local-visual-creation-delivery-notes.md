# 本地视觉创作交付说明

## 2026-06-13 Stage 1.7

本版本修复 Stage 1.6 的严重退化：8 步采样结束后进入 1131 / 1131 的 VAE tiling 解码进度并长时间卡死，且生成期间返回按钮被锁定。

### 本版本修复内容

- SD1.5 快速验收 native 推理关闭 VAE tiling，避免 256 x 256 验收图进入 1131 个瓦片的 CPU 解码循环。
- 进度回调只接受总步数等于当前采样步数的进度，过滤 VAE tile progress，不能再显示 1131 / 1131 为生成步数。
- 采样完成后仍显示“正在解码并保存图片”，但不再被瓦片进度污染。
- 生成期间不再禁用顶部返回，用户不用强制结束应用才能退出页面。

### 手动测试方式

1. 覆盖安装本版本 APK。
2. 使用 Stable Diffusion 1.5 Q4_0 GGUF 生成同一条英文提示词。
3. 确认进度只在 0 到 8 步范围内变化，不再出现 1131 / 1131。
4. 采样到 8 / 8 后等待解码保存，确认不会卡住两小时。
5. 生成期间尝试返回，确认页面不再锁死。

## 2026-06-13 Stage 1.6

本版本修复 SD1.5 生成完成后 PNG 预览碎片化、采样完成后状态文案回到“正在加载模型”、以及保存按钮不可用的问题。

### 本版本修复内容

- native 推理结果不再只返回像素字节，而是同时返回 stable-diffusion.cpp 实际输出的宽度、高度和通道数。
- PNG 保存逻辑改为按 native 实际宽高写入，避免底层返回 512 x 512 时被 Kotlin 按 256 x 256 错误解释成碎片图。
- 采样达到总步数后，状态文案改为“采样已完成，正在解码并保存图片”，不再误导为重新加载模型。
- “保存到相册”按钮在生成图片后可点击，保存到系统相册的 Pictures/Local Visual Creation。
- “保存到本地文件夹”按钮在生成图片后可点击，保存到下载管理的 local-visual-creation/generated-images。

### 手动测试方式

1. 覆盖安装本版本 APK。
2. 使用已经下载的 Stable Diffusion 1.5 Q4_0 GGUF 进入视觉创作页面。
3. 输入英文提示词并点击“生成图片”。
4. 等采样进度跑到 8 / 8 后，确认状态进入解码保存，而不是重新显示加载模型。
5. 生成完成后确认预览图不再是碎片化错行图片。
6. 点击“保存到相册”，再到系统相册确认图片可见。
7. 点击“保存到本地文件夹”，再到下载管理的 local-visual-creation/generated-images 确认图片可见。

### 成功标准

- 生成结果区显示 PNG 路径和正常预览。
- 保存到相册按钮可点击，并能让系统相册看到图片。
- 保存到本地文件夹按钮可点击，并能在下载管理目录找到图片。
- 采样完成后的等待阶段不再显示“正在加载模型”。

## 2026-06-13 Stage 1.5

本版本修复点击生成后长时间只显示“正在生成图片”且用户无法判断是否已经提交、是否正在加载模型、是否进入采样的问题。

### 本版本修复内容

- 点击“生成图片”后会清空输入框，并在页面新增“已提交的生成请求”，明确显示已经提交的提示词和负面提示词。
- 状态区改为先播报“已提交提示词”和模型加载耗时，每 10 秒刷新一次加载等待时间。
- JNI 接入 stable-diffusion.cpp 的 progress callback，进入采样后会显示第几步和总步数。
- JNI 在模型加载后检查 `sd_ctx_supports_image_generation`，如果模型不能文生图，会直接给出明确错误。
- Stable Diffusion 1.5 工作台默认切换为 256 x 256、8 步 CPU 快速验收配置，避免手机首次测试直接跑 512 x 512、20 步导致长时间不可感知等待。

### 手动测试方式

1. 覆盖安装本版本 APK。
2. 进入已经下载的 Stable Diffusion 1.5 Q4_0 GGUF 工作台。
3. 确认高级参数中尺寸为 256 x 256，采样步数为 8。
4. 在提示词输入框输入英文提示词，负面提示词可留空。
5. 点击“生成图片”。
6. 确认输入框清空，页面出现“已提交的生成请求”，状态区开始播报加载耗时。
7. 等进入采样后，确认状态区或按钮文字显示第几步和总步数。
8. 等待生成完成，确认生成结果区显示 PNG 路径和图片预览。

### 成功标准

- 点击生成后用户能明确听到提示词已经提交。
- 模型加载阶段不是静止的一句话，而是会刷新等待耗时。
- 采样阶段能听到第几步和总步数。
- SD1.5 Q4_0 至少能以 256 x 256、8 步完成一次 PNG 生成。
- 如果底层模型无法文生图，页面显示明确错误，不再无限“正在生成”。

## 2026-06-12 Stage 1.4

本版本修复工作台模型显示不一致和页面元素混乱问题，并接入第一版 stable-diffusion.cpp native 推理。

### 本版本修复内容

- 工作台当前模型改为读取模型管理页实际选中的模型，不再默认显示列表第一个 Z-Image Turbo Q2_K。
- 工作台不再直接铺开 10 个图像生成模型；模型下载、切换和删除仍在上一层模型列表完成。
- 高级生成参数默认收起。
- 图片后续处理区域默认收起。
- 生成按钮接入 stable-diffusion.cpp JNI native 推理。
- 本版本真实推理先支持 Stable Diffusion 1.5 单文件 GGUF 模型，包括 SD1.5 Q4_0、Q5_0、Q8_0。
- 生成结果保存为 PNG，目录为应用外部文件目录下的 `visual_creation_outputs`。

### 用户应该能体验到的功能

- 下载并进入 Stable Diffusion 1.5 Q4_0 GGUF 后，工作台当前模型应显示 Stable Diffusion 1.5 Q4_0 GGUF。
- 工作台页面不应再出现 10 个模型芯片列表。
- 输入提示词和负面提示词后，点击“生成图片”会进入真实生成状态。
- 生成完成后，页面应显示生成图片文件路径，并展示图片预览。

### 手动测试方式

1. 覆盖安装本版本 APK。
2. 进入“本地视觉创作”。
3. 确认模型列表仍显示 10 个模型。
4. 选择已经下载的 Stable Diffusion 1.5 Q4_0 GGUF，点击试试看。
5. 确认工作台当前模型显示 Stable Diffusion 1.5 Q4_0 GGUF。
6. 输入英文提示词，例如 `a violinist under street light, cinematic, realistic`。
7. 输入负面提示词，例如 `blurry, low quality, deformed`。
8. 点击“生成图片”，等待首次模型加载和推理完成。

### 成功标准

- 当前模型显示与用户点击进入的模型一致。
- 工作台主页面不再直接显示 10 个模型列表。
- SD1.5 Q4_0 能真实生成 PNG 文件。
- 生成失败时应显示明确错误，不应崩溃。

### 当前限制

- Z-Image Turbo 量化模型仍只作为可下载候选列出，本版本 native 推理先不支持 Z-Image。
- 保存到系统相册、发送给视觉语言模型处理、自动模式仍在后续阶段。

## 2026-06-12 Stage 1.3

本版本扩展“本地视觉创作”的可下载图像生成模型列表，并继续沿用固定 release 签名和递增测试版本号。

### 本版本修复内容

- 模型列表从 1 个扩展到 10 个。
- 保留已经下载过的 `z-image-turbo-q2-gguf` 作为第一推荐项，模型名和本地版本目录保持不变。
- 新增 Z-Image Turbo Q3_K、Q4_0、Q4_K、Q5_0、Q6_K、Q8_0。
- 新增英文 Stable Diffusion 1.5 Q4_0、Q5_0、Q8_0。
- 每个模型条目都写入真实 Hugging Face 下载 URL 和核验过的文件大小。

### 用户应该能体验到的功能

- 首页“本地视觉创作”入口应该显示包含 10 个模型。
- 进入模型页后，应该能听到或看到多个 Z-Image Turbo 量化版本。
- 已下载的 Z-Image Turbo Q2_K 仍应被识别为已下载。
- 用户可以自行选择下载更高质量的 Q3、Q4、Q5、Q6、Q8 版本，或选择英文 SD1.5 候选模型。

### 注意事项

- 当前下载框架会把不同量化版本放在各自模型目录中，因此下载其他 Z-Image 版本时会再次下载该版本所需的 VAE 和文本编码器文件；后续会继续优化共享文件复用。
- 本版本仍未接入 native stable-diffusion.cpp 推理引擎，模型列表扩展是为下一步推理引擎接入做准备。

## 2026-06-12 Stage 1.2

本版本先修复测试链路的基础问题：固定云端 release 签名，并让测试版本号自动递增。

### 本版本修复内容

- 云端构建不再使用每台 runner 临时生成的 debug 签名。
- GitHub Actions 改为从仓库 Secrets 注入固定 release keystore。
- `versionCode` 改为云端自动递增，避免多个 APK 都是 `100`。
- `versionName` 改为 `1.0.13-plaza.visual.<run_number>`，便于屏幕阅读器区分测试版本。
- 云端构建会在上传 APK 前打印签名证书和 APK 版本信息，避免再次交付未知签名包。
- 已将当前已下载的 Z-Image Turbo Q2_K 模型包备份到 `/storage/emulated/0/下载管理/local-visual-creation/model-backup/`。

### 重要限制

上一版已经安装到手机上的 APK 使用了临时 debug 签名，私钥无法从 GitHub runner 找回。因此第一次切换到固定 release 签名时，Android 仍然不能覆盖安装上一版。完成这一次签名切换后，后续所有固定 release 签名版本才可以覆盖更新。

### 手动测试方式

1. 安装本版本前，确认模型备份目录仍存在。
2. 卸载上一版临时 debug 签名 APK。
3. 安装本版本固定 release 签名 APK。
4. 将备份模型目录还原到 `/storage/emulated/0/Android/data/com.localagent.plaza/files/z_image_turbo_q2_gguf/`。
5. 打开应用，确认“本地视觉创作”仍显示 1 个模型。
6. 进入模型页面，确认模型不需要重新下载或能识别已下载文件。

### 成功标准

- 本版本 APK 的签名证书 SHA256 固定为 `38:A9:A4:F1:5E:D5:3F:47:AB:EE:1A:03:43:B2:FE:3D:82:56:87:AC:B1:48:AC:8C:52:2F:A1:D2:9F:3E:29:2D`。
- APK 的 `versionCode` 大于 100。
- 安装完成后，下一次再产出的 APK 可以覆盖更新这个版本。
- 备份模型文件可以还原，避免再次下载 6GB 模型包。

## 2026-06-12 Stage 1.1

本版本修复上一版“本地视觉创作”入口显示包含 0 个模型且无法进入的问题。

### 本版本修复内容

- 删除 `workbench.marker` 占位模型，不再用空入口冒充模型。
- 接入真实可下载的 Z-Image Turbo Q2_K GGUF 图像生成模型包。
- 模型包包含 3 个公开可下载文件：
  - `z_image_turbo-Q2_K.gguf`，来源 `leejet/Z-Image-Turbo-GGUF`，大小 2,592,442,304 字节。
  - `ae.safetensors`，来源 `Comfy-Org/z_image_turbo`，大小 335,304,388 字节。
  - `qwen_3_4b_fp4_mixed.safetensors`，来源 `Comfy-Org/z_image_turbo`，大小 3,479,416,193 字节。
- 修复模型管理器初始化时清空非 allowlist 模型导致视觉创作任务变成 0 个模型的问题。
- 修复带附属文件模型下载总大小被重复计算的问题。
- 模型列表不再给图像生成模型显示 LLM benchmark 按钮。

### 用户应该能体验到的功能

- 首页“本地视觉创作”入口应该显示包含 1 个模型。
- 点击“本地视觉创作”应该进入模型列表，而不是无反应或自动返回。
- 模型列表应该显示 “Z-Image Turbo Q2_K GGUF”。
- 展开模型卡片应该能看到模型来源、格式、后端、文件总大小、推荐尺寸和许可证说明。
- 点击下载应该开始下载主模型、VAE 和文本编码器这 3 个真实文件，并显示下载进度。
- 下载成功后点击进入应该打开本地视觉创作页面。
- 创作页面应该显示提示词、负面提示词、生成参数、生成结果区、保存按钮、发送给视觉模型处理按钮和图片后续处理区。

### 手动测试方式

1. 安装本版本 APK。
2. 打开应用首页，听读或查看“本地视觉创作”入口的模型数量。
3. 点击“本地视觉创作”入口。
4. 在模型列表中找到 “Z-Image Turbo Q2_K GGUF”。
5. 点击下载，确认下载进度开始变化。
6. 等待下载完成后点击进入工作台。
7. 在工作台输入提示词和负面提示词，点击“生成图片”。

### 成功标准

- 首页显示“本地视觉创作”包含 1 个模型。
- 入口可以打开模型列表。
- 模型列表中不是占位模型，而是真实的 “Z-Image Turbo Q2_K GGUF”。
- 下载按钮可以触发真实下载，下载进度总大小约 6.41GB。
- 下载完成后可以进入视觉创作页面。
- 点击“生成图片”时，应用明确提示 native 图像生成推理引擎尚未接入，不崩溃、不假装生成成功。

### 失败标准

- 首页仍显示包含 0 个模型。
- 点击“本地视觉创作”无反应或自动返回。
- 模型列表里出现 `workbench.marker` 或“本地视觉创作工作台”占位模型。
- 下载立即失败且不是网络中断、存储不足或 Hugging Face 限流造成。
- 下载进度总大小明显不是约 6.41GB。
- 下载完成后无法进入视觉创作页面。

### 当前未完成内容

- stable-diffusion.cpp native Android 推理引擎尚未接入，因此本版本不算真实文生图验收版本。
- 生成图片后的保存到相册、发送给视觉语言模型描述、自动模式串联推理仍是下一阶段工作。

# 视觉旁白 · Vision Narration

> 把 Android 设备的相机实时画面，翻译成你"听得到"的描述。

视觉旁白是 Local Agent Plaza 的差异化亮点能力。它把本地多模态模型、TTS 引擎、相机采集和提示词工程串成一条无障碍工作流，让用户不必盯着屏幕也能"看到"世界。

## 设计目标

- **完全本地**：从图像采集到语音播报，所有推理与播放都不离开设备
- **低延迟**：典型工作流下首帧播报 < 3 秒
- **可定制**：提示词、播报频率、历史记录全部用户可控
- **无障碍优先**：对视障用户、视疲劳用户、双手被占用的场景尤其友好

## 工作流

1. **相机抓帧**：以可配置的时间间隔从后置相机获取一帧图像
2. **本地多模态推理**：调用本地 LLM（必须支持图像输入），结合用户自定义的提示词生成描述
3. **TTS 播报**：把生成的描述用系统 TTS 或本地 TTS 引擎朗读
4. **历史记录**：所有旁白条目按时间戳存档，可导出为文本或 Markdown

## 核心配置项

在应用的"视觉旁白"设置中可以调整：

| 配置 | 默认值 | 说明 |
|---|---|---|
| 抓帧间隔 | 5 秒 | 两次相机采样之间的最小间隔 |
| 模型 | 已选的多模态模型 | 推理用的本地模型 |
| 自定义提示词 | "请描述当前画面" | 每次推理前拼接的指令 |
| 播报开关 | 开启 | 是否启用 TTS 朗读 |
| 语音 | 系统默认 | 选 TTS 引擎与音色 |
| 语速 | 1.0x | 0.5x ~ 2.0x 可调 |
| 历史保留条数 | 50 | 超过后滚动丢弃最旧条目 |

## 提示词示例

- "用一句话告诉视障用户面前最值得关注的内容"
- "描述这张图片里的文字、路牌和方向"
- "我手里拿着什么？周围有几个人？环境亮度如何？"
- "把当前画面翻译成日语"

提示词越具体，模型输出越稳定。复杂任务建议拆成多个简短指令。

## 历史与导出

- 历史记录按时间倒序展示
- 支持导出为纯文本（含时间戳）或 Markdown
- 导出文件保存到本地文件工作区，可通过分享/邮件发送
- 历史记录在卸载应用时一并清除，**不会上传任何地方**

## 性能与电量建议

- 抓帧间隔建议 ≥ 3 秒，避免设备过热
- 关闭"持续在后台运行时"可显著延长待机时间
- NPU 加速的多模态模型（如 Gemma 3n）能带来数倍速度提升
- 在弱光场景下，建议把"环境亮度"加进提示词，让模型知道当前画面质量

## 适用场景

- **视障辅助出行**：把街道、商店、路牌朗读出来
- **儿童陪伴阅读**：把绘本画面翻译成口述故事
- **工业巡检**：在双手被占用时由 AI 描述设备状态
- **学习/工作场景**：快速记录视线范围内的关键信息

## 已知边界

- 仅在多模态模型已下载并加载后才可启动
- 强光 / 运动模糊 / 极端角度可能让模型输出不稳定
- TTS 受系统引擎限制，部分设备上无法调整音色
